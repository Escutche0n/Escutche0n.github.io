### 接下来很想填的坑

##### 引子

最近真的是有好多东西想写，我猜是刚尝到甜头的缘故吧，似乎是找到了那种当少数派作者执笔的愉悦感，脑子里始终有一种奇怪的想法，就是如果不断的练习加上有这么多素材可以拿来填充自己的生活那必然是极好的。

> **可是当真正开始的时候，在明确了自己并非出于懒的前提之下，我会慢慢回过头来审视自己，这一切是为了什么？**

掰掰指头算算近几日脑子里想写的几个主题：

1. 告诉小伙伴们我是如何完成这样一个**对我自己而言**浩大的工程的
2. 记录**自己**如何**对我自己而言**科学地瘦身
3. 记录**自己**如何用 **Sous Vide** 做出美味的食物
4. 告诉阿粥如何**更好地**使用电脑浏览器
5. ... ...

##### 第一个大坑

细分下来的话，目标可能就更明确了：

1. 写一篇关于 **Markdown** 的语法入门指南，并推荐 **Typora** 等 Markdown **编辑器**。
2. 写一篇关于如何使用 **GitHub Pages** 在云端存储内容。
3. 写一篇关于 **Jekyll** 的指南，讲解 Jekyll 如何将 MD 这类的**平文本转化成网页**。
4. 写一篇关于这个英文原版 **BootStrap** 主题模版，黄玄的中文模版，邱柏荧的简化版模版。
5. 写一篇关于我自己的修改过程，如何自定义**照片墙**页面。
6. ……

至此终于写完了第一个大坑，接着再去完成第二个第三个第四个，可是真的有必要吗？

作为一个天天刷少数派的同学，先不说我看过多少 ( 基本除了**每周新app**推荐之外的都会看，我自己也是两个 Apple ID 上加起来有几千个app的人新app就算不看少数派也能感知到不少 )，单单是平台上提及 Markdown的文章有82篇可查，其中入门向的:

- [想学 Markdown？这篇文章帮你快速上手](https://sspai.com/post/43866)
- [Markdown快速入门](https://sspai.com/post/45816)
- [Markdown 完全入门（上）](https://sspai.com/post/36610)
- [Markdown 完全入门（下）](https://sspai.com/post/36682)
- [认识与入门 Markdown](https://sspai.com/post/25137)
- [少为人知的Markdown基础知识](https://sspai.com/post/37273)

这些标题感受下，内容的重复率有多高我不想赘述，只是应证了一句再正常不过的话：

> 这个时代最平凡的一件事就是自命不凡了。

反正大概是这意思，就是说这个在人人追求独特标榜态度的时代，不想要变得独一无二反而是个稀罕事儿，并非要去说他们不好，只是我一个庸俗的人去抱着一个态度，自己去写一篇让朋友看得懂的 Markdown 教程，**说是委屈了朋友看一篇低劣的文章，倒不如说是委屈了自己**。细心向他们说这些资源是极好的，推荐给他们，有问题再来问自己并耐心解答尚可，对嘛。

就我个人而言，更喜欢那些**思考类**的文章，例如这个 [反思Markdown：Markdown的长与短](https://sspai.com/post/37340)，虽然没有挨着挨着截图说这是啥那是啥，但**深度的思考才能真正让我们去了解一门技术背后的逻辑晓其优劣并将之作为我们创作的工具而并非束缚我们的思维**，这也是为什么在 Git 入门那篇文章里我想侧重它是用来简化版本的缘故，因为自己原来管理 Word 版本就笨的是很混乱但 Markdown + Git 实实在在从技术上解决了这一个困扰。

话说回来，当小伙伴们掌握了 Markdown 的语法之后，读邱柏荧大大的 [GitHub Pages 快速搭建个人博客](https://www.jianshu.com/p/ee0826c68d19) 大概就能知道个十之七八了，此时再回过头来看 [Hux Blog](https://github.com/huxpro/huxpro.github.io/) 就能大致了解了。虽然黄玄大大也给出了简洁的模版但实际上这个就是邱柏荧同学的模版来源，HUX 的技术水平应该是比 BY 要更高阶的，代码更干净，但 BY 的中文指南会更细致更有好一些。总之看完这些，第一个大坑大致就填上了。



##### 剩下的坑

所以其实这样思考下来，我真正需要揣摩的，是那些吸引我的地方，直言不讳说是内心和思想吧，反正这是我的地盘对不对想怎么写怎么写。Anyway 开个玩笑啦，但一个人真正吸引我的，并不单单只是故事，这些经历并不足以客观的去反应一个人的方方面面，但你读过的书，走过的路，做过的事，才是让我为之倾倒的吧，我想，所以脑子里那些天马行空的东西，才是我真正要自己留下的。

如何减肥并非告诉看客们如何去控制卡路里，如何去锻炼，而是我背后的思考，所以这个坑有空我靠晚上会好好填，之前我有一个很神奇的算法等空了会截图做出来，百分百能瘦，看过一本 **Ray Dalio** 的叫[**原则**](https://book.douban.com/subject/27608239/)的书里面有一条观点有相似，所以分享出来可以给各位以启示。

Sous-Vide 则是我如何用科学的办法，去尝试餐厅级别的美食的图片日志，就当是写完少数派写下厨房吧，哈哈。

> **至于剩下的时间，我想留给脑子，去思考，留给镜头，去 capture，留给身体，去运动，去旅行，留给豆瓣，补完我500部电影的坑哈哈。**岁月会把我磨成自己想要的样子的，我想。

以上。
