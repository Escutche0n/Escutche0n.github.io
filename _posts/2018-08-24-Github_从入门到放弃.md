> 不管影视作品之中的技术肥宅是如何描述的，一个软件开发者的生活绝对不包括久坐在一个暗无天日的房间里，其键盘仅靠着屏幕微弱的光照亮着，与他人交流这种想法于其不修边幅的生态来讲也是恶心至极。不不不不不，那是肥宅，不是技术，一个软件开发者是社交和活动型的生物，也会时常对生活感到绝望和焦躁，上帝说，倘若时光倒流，于是我们便有了 git 和 Github.



简而言之，Git 是一个版本控制系统，它能用来跟踪文件版本的变动，并让一个团队的成员之间合作起来容易很多，前提是大家对这个工具有最基本的了解。可是在接触之前，是的，它很吓人，所以我在这里记录一些自己作为初学者的心得和感受。



### 什么是 Git?

如同之前所说，Git 是一个版本控制系统，是由一个叫[林纳斯·托瓦兹](https://en.wikipedia.org/wiki/Linus_Torvalds)的人创建的，对了，这位大佬还发明了 Linux。不过我们可不需要运行 Linux 来使用 Git 工具，用 Windows 和 Mac 即可。

Git 允许你保持对文件过去每一个版本的持续追踪，不管是你还是你的团队成员编写的版本，这有什么意义呢：

> 期末论文快要交了，我打开了一篇论文，有如下命名：阿陈的Essay.docx，阿陈的Essay初版.docx，阿陈的Essay修改版.docx，阿陈的Essay最终版反馈修改版.docx...

这还只是阿陈一个人纠结命名的情况，倘若和阿粥合作的话，还不包括诸如“阿陈的Essay最终版反馈修改版经过阿粥校对版”，难道不能有一个一劳永逸的办法吗，一个存在服务器上的版本，我在需要的时候从上面下载下来最新的内容，完成修改，并告诉服务器：“阿粥今天校对了”，上传，搞定。







**The basic mechanism is**

1. Initialise the local folder
2. Clone / Pull files from repository to the folder
3. Edit
4. Commit comments for this edit
5. Push the files back to the repository on the server.
