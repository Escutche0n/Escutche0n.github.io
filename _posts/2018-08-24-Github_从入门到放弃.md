### 什么是 Git?

如同之前所说，Git 是一个版本控制系统，是由一个叫[林纳斯·托瓦兹](https://en.wikipedia.org/wiki/Linus_Torvalds)的人创建的，对了，这位大佬还发明了 Linux。不过我们可不需要运行 Linux 来使用 Git 工具，用 Windows 和 Mac 即可。

Git 允许你保持对文件过去每一个版本的持续追踪，不管是你还是你的团队成员编写的版本，这有什么意义呢：

> 期末论文快要交了，阿陈打开了一篇论文，有如下命名：**阿陈的Essay.docx**，**阿陈的Essay初版.docx**，**阿陈的Essay修改版.docx**，**阿陈的Essay最终版反馈修改版.docx**...

这还只是阿陈一个人纠结命名的情况，倘若和阿粥合作的话，还不包括诸如 “**阿陈的Essay最终版反馈修改版经过阿粥校对版**”，难道不能有一个一劳永逸的办法吗，一个存在服务器上的版本，我在需要的时候从上面下载下来最新的内容，完成修改，并告诉服务器：“**阿粥今天用她的 Thinkpad 校对了**”，上传，搞定。

而这正是 Git 能为我们做的，一个跨平台的强大工具，无论你身处何处，无论你用的是 **Windows** 还是 **Mac**，你都能一定程度去完成手上的工作，倘若阿陈对阿粥的这个校对不满意，还能轻轻松松回到修改前的版本。而类似于 **[GitHub](https://github.com/)** 和 **[BitBucket][https://bitbucket.org/]** 这样的平台，则是我们远程的服务器所在之处。

> 阿陈哪敢对阿粥的修改不满意哈哈哈哈哈哈哈哈，Fine！

总之一句话，如果你还没有使用 Git，那就从现在开始入门吧。



### **Git 的核心机制**

1. **初始化** (**Init**) 本地文件夹
2. 从**仓库** (**Respository**) **复制** (**Clone**) 文件到本地文件夹
3. **编辑** (**Edit**)文件
4. 为这个编辑做个**陈述** (**Commit**)
5. 将文件上传到服务器里的仓库并**比对** (**Fetch**)

其中和服务器之间下载和上传的这个过程则更生动称为**拉** (**Pull**)和**推** (**Push**)。



### Git 的示例代码

假设阿陈的仓库地址为：https://github.com/Escutche0n/HelloGit.git

```haskell
cd desktop								-- 定位到桌面
git init 								-- 初始化
git clone https://github.com/Escutche0n/HelloGit.git	--复制文件到本地
git add.								-- 添加
git commit -am "[query]"						-- 陈述添加的内容
git pull								-- 发回至云端
```
